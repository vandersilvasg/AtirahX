# 00_SCRIPTS_COPY_PASTE_GUIA

## Objetivo
Padronizar uso dos scripts em duas opcoes:
1. Versao curta: mais rapida para iniciar.
2. Versao completa: mais controle, escopo maior e menos ambiguidade.

## Regra de Leitura
- Arquivos `*_SHORT.md` = versao curta.
- Arquivos sem `SHORT` = versao completa.

## Recomendacao
1. Use versao curta para prototipo rapido.
2. Use versao completa para implementacao de producao.

## Ordem sugerida
1. Lovable (prototipo visual)
2. Cursor ou Claude Code (implementacao)
3. Gemini/Antigravity (refino, planejamento, automacao adicional)
